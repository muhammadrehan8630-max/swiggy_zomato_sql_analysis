# 🚴 Swiggy & Zomato Delivery Operations Analysis — SQL + Power BI

## 📌 Project Overview
This project analyzes Swiggy and Zomato food delivery operations using PostgreSQL and Power BI.

The analysis focuses on:
- Delivery performance
- Rider efficiency
- Cancellation trends
- Peak order hours
- Operational bottlenecks

The goal is to generate business insights that can help improve delivery operations and customer experience.

---

## ❓ Business Questions Answered
- What are the peak order hours?
- What is the overall cancellation rate?
- Which riders have the highest average delivery time?
- Which time periods experience higher cancellations?
- How does delivery time vary across different days?
- Which riders handle the highest order volume?

## 🛠️ SQL Concepts Used
- Aggregations with GROUP BY and HAVING
- Subqueries (correlated and nested)
- CASE WHEN statements
- COALESCE and NULLIF for data safety
- EXTRACT function for hourly analysis
- Date and time calculations
- Multi-condition filtering
- Window Functions (RANK, LAG)

---

## 🔍 Key Findings
- Peak order activity occurs during afternoon and evening hours.
- Overall cancellation rate is approximately 1.16%.
- Average delivery time is around 32 minutes.
- Certain riders show consistently higher delivery times.
- Cancellation spikes are visible during specific operational hours.
